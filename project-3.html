Project 3 - Network Security Lab: VLANs, DHCP, and Layer 2 Security

Developed a secure network infrastructure with VLAN segmentation, allocating IPs through a DHCP server and controlling access via DHCP snooping. Added multiple security layers such as port security and storm control on switches to prevent network flooding and unauthorized device connections. Enhanced resilience with an EtherChannel setup for redundancy, ensuring consistent network performance even in failure scenarios.

Step 1: Basic Setup – Two VLANs created: Guest and Employee.

Responding to the client’s request for secure, compartmentalized access, we established two VLANs: 'Guest' for external users and 'Employee' for internal staff. This configuration channels guest traffic through a secure path, isolated from employee workflows, reducing the risk of unauthorized access to sensitive data. The setup not only enhances security but also organizes network traffic for efficient management, setting the stage for advanced control measures like DHCP Snooping and access policies in later stages.

We implemented VLANs for 'Guest' and 'Employee' and set up a DHCP server to manage IP addresses dynamically. After getting the devices configured, I ran some pings to test connectivity—and hit a roadblock as the pings failed. Instead of panicking, I analyzed each configuration step, from IP addressing to VLAN assignments. By fine-tuning the DHCP settings and verifying VLAN trunking, I addressed the core issues, ensuring the network was stable and ready for future scaling.

After establishing basic network connectivity and configuring DHCP to automate IP addressing, I implemented DHCP snooping to secure the network against rogue DHCP servers. This feature was applied across both VLANs—Guest and Employee—providing an additional layer of security by ensuring only legitimate DHCP offers were allowed. With this in place, each VLAN maintained protected and isolated DHCP traffic, reducing unauthorized access points.

To further strengthen network security, I enabled port security on the switch, limiting the number of MAC addresses per port to prevent unauthorized device connections and mitigate the risk of MAC address spoofing. Each active port was restricted to allow only two unique MAC addresses, adding a simple but effective security mechanism.

Additionally, I implemented Link Aggregation Control Protocol (LACP) between the two core switches. By combining multiple Ethernet links into a single logical connection, LACP created redundancy, helping to eliminate single points of failure. This setup ensures that if one link fails, traffic will continue to flow uninterrupted over the other link, improving the network’s resilience and reliability. With DHCP snooping, port security, and LACP in place, the network was fortified with robust safeguards against common vulnerabilities, establishing a foundation for secure, scalable infrastructure.
